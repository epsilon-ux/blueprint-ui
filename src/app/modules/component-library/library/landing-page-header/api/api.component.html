Selector
bp-landing-page-header

Inputs

Landing page header inputs
Input Type Description Notes
headingText string The landing page's title. Required.
actionText string The action's text. Required when action is present e.g. bpRouterLink/href input or actionClick output is provided.
dropdownItems object[] An array of objects used to create the items in the dropdown. See Dropdowns API for more information on what should be included in each object.
bpRouterLink string The action's routerLink attribute value. Cannot have both bpRouterLink and href.
href string The action's href attribute value. Cannot have both bpRouterLink and href.
isBottomMargin boolean Determines if there is a bottom margin.

Default is "true".

When "false" the landing page header does not have a bottom margin.
bpID string The value for the landing page header's id attribute.

Ids must only be used once per page.

Ids should be camelCase, e.g.: #myId.

See identifiers.

Outputs

Landing page header outputs
Output Description
actionClick Emits an event on click of the primary button or dropdown menu.

Content Projection

The optional ng-content directives' select attribute value is tab-items.

tab-items

Non-navigational tabs

Non-navigational tabs should use button elements supplemented by aria attributes in order to programmatically associate the tab with its corresponding tab pane.

Navigational tabs

Navigational tabs should use anchor elements with routerLinks for internal links or hrefs for external links. The tabs should be wrapped in a nav element, and the nav element's aria-label value should reflect the level of the navigation in context, e.g.: secondary navigation, tertiary navigation, etc.
