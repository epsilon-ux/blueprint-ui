Inputs

Table inputs
Input Type Description Notes
data object[] An array of objects where each object represents a row in the table. Required.
isDataLoading boolean Indicates if the table data is loading.

If "true", the loading icon and "Loading data" message renders in the table.

If "false" and there is no data, the "No data available" message renders in the table.

properties object An object used to configure an instance of the table component.

Use the properties object to enable or disable configurable features.

See the Properties table for more information.

Properties

Table properties
Property Type Description Notes
caption string A sentence used for explaining the table to people using a screen reader. The text is only visible to screen readers. Required.
rowId string The key from the data to use as the identifier.
columns Column[] An array of objects that define the columns in the table. Columns will render in the table in the same order as in this array. Required.
sort.defaultSortedColumn string The key from the data that represents the column that is sorted by default / on page load. The column that is sorted by default cannot be hidden with the column selector.
expandableRowsTemplate TemplateRef Template for the content of the expandable row.

Expandable rows feature is enabled when a template is passed in.

See Expandable Rows

hasSelectableRows boolean Enables the selectable rows feature. Use "true" to enable the selectable rows feature, and render checkboxes in the first column of each table row.
actions Action[] An array of objects where each object represents a button in the actions column. When there are more than 3 actions, the individual buttons collapse into a dropdown.
hasColumnSelector boolean Enables the column selector feature. Use "true" to enable the column selector, and render the column selector below the table.
hasDisplayDensity boolean Enables the display density feature.

Use "true" to enable the display density feature, and render the display density selector below the table.

The default display density is "comfortable."

The user can select "compact" to decrease the padding of the table's rows.

pagination.hasPagination boolean Enables the pagination feature.

Use "true" to enable pagination, and render the pagination below the table.

Pagination includes: the total number of results, a select that allows the user to select the number of rows to display per page, and pager buttons.

internationalization object Contains all the strings of text used in the table component. See internationalization for more details.

Columns

Columns is an object that is part of the properties Input.

Column properties
Property Type Description Notes
key string The key from the inputted data to use for data in the corresponding column. Required.
headerText string The text to display in the corresponding column header. Required.
isColumnDisplayed boolean Indicates whether or not the column should be displayed by default. When "true" the column is displayed by default.
type enum ColumnType {
ID,
TEXT,
LINK,
DATE,
BOOLEAN,
INPUT,
STATUS,
ICON,
DROPDOWN
} 		The type of the column. This determines how to handle the content.
isSortable boolean Indicates if the column is sortable.

Default is "true".

When "true" the column is sortable.
sortFunctionAsc Function The function passed into the sort function when sorting the data by the column in ascending order (small to large).

The default sort function takes the difference of the first and second elements to determine sort order.

The function should be formatted like a compare function, including having two parameters.

The function will be used and then the array will be reversed for descending sort.
template TemplateRef Pass in an ng-template that will be used for the content in this column. Required when column type is TEMPLATE.
icon object An object containing information for when and how to render icons in an icon column.

Required when column type is ICON.

See Icons for more details.
link object An object containing the link details.

Required when column type is LINK.

See Links for more details.
statusIndicatorMapping object Maps the standardized class names for status indicators to the product specific text. Required when column type is STATUS. The available classes for mapping are incomplete-primary, incomplete-secondary, in-progress, warning, complete. For more information on status indicators see Zeplin.

Icons

Icon properties
Property Type Description Notes
color string The color of the icon. Expects either "midnight" or "warning".
mapping object Maps icons to text. The key is the text that represents the icon, the value is the class name for the Font Awesome icon.

Links

Link properties
Property Type Description Notes
element string The element used for the link.

Expects either "a" or "button".

Use "a" when the link column should route to another part of the application.

Use "button" when the link column should perform an action on the page, e.g. opens a modal.

ariaLabel string The format of the aria-label to be used for the link. To use values from the row, insert #{key} anywhere into the string where "key" is the name of the data property to insert into the label, e.g. "Open row #{id}" NOTE: These are still regular strings, not javascript template strings, so use single/double quotes.
target string The string to be used in the link's target attribute. Required when element = "a". "_blank" opens in a new tab.
path string The string to be used in the link's routerLink attribute. One of either path or href is required when element = "a".
href string The string to be used in the link's href attribute. One of either path or href is required when element = "a".
action string The string to be emitted when the button is clicked. Required if element = "button". See Outputs for more information on what is emitted.

Actions

Actions is an object that is part of the properties Input.

Action properties
Property Type Description Notes
element string The element used for the action.

Expects either a or button.

Use a when the action should route to another part of the application.

Use button when the action should perform an action on the page, e.g. opens a modal.

text string The text to be used in the action button/link.
ariaLabel string The format of the aria-label to be used for the button/link. To use values from the row, insert #{key} anywhere into the string where "key" is the name of the data property to insert into the label, e.g. "Open row #{id}" NOTE: These are still regular strings, not javascript template strings, so use single/double quotes.
class string The class(es) to be used on the action button/link

E.g. btn-outline-primary

If element = "button", .btn will be included by default.
target string The string to be used in the link's target attribute Required when element = "a". E.g. "_blank" opens in a new tab.
routerLink string The string to be used in the link's routerLink attribute. One of either path or href is required when element = "a".
href string The string to be used in the link's href attribute. One of either path or href is required when element = "a".
action string The string to be emitted when the button is clicked. Required if element = "button". See outputs for more information on what is emitted.
conditions (object | string)[] The conditions used to determine if the action will render. Conditions is an array containing objects and strings. The condition objects have compare operator strings between them.

Outputs

Output Type Description

selectedRowsAction array An array of the currently selected rows is emitted whenever a change is made to the selected rows i.e. one or all row(s) is select or deselected.

Action output object

Action properties

Template Columns

Templates can be provided to specified columns to allow custom columns.

  1. Create a template in the component. This should either be a sibling or a child of the table component.
  2. Select the template in the component typescript.
  3. Pass the template into a column object of type "TEMPLATE".

Expandable Rows

When enabled, each row has an expandable row using a template passed in from the properties object.

  1. Create a template in the component. This should either be a sibling or a child of the table component.
  2. Select the template in the component typescript.
  3. Pass the template into the expandableRowsTemplate via the properties Input object.
