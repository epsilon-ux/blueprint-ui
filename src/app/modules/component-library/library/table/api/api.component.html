Inputs

Table inputs
Input Type Description Notes
data object[] An array of objects where each object represents a row in the table. Required.
dataLength number The total number of rows in the entire dataset the table can paginate through. Required.
isDataLoading boolean Indicates if the table data is loading.

If "true", the loading icon and "Loading data" message renders in the table.

If "false" and there is no data, the "No data available" message renders in the table.

properties object An object used to configure an instance of the table component.

Use the properties object to enable or disable configurable features.

See the Properties table for more information.

Properties

Table properties
Property Type Description Notes
caption string A sentence used for explaining the table to people using a screen reader. The text is only visible to screen readers. Required.
rowId string The key from the data to use as the identifier.
columns Column[] An array of objects that define the columns in the table. Columns will render in the table in the same order as in this array. Required.
sort.defaultSortedColumn string The key from the data that represents the column that is sorted by default / on page load. The column that is sorted by default cannot be hidden with the column selector.
expandableRowsTemplate TemplateRef Template for the content of the expandable row.

Expandable rows feature is enabled when a template is passed in.

See Expandable Rows

hasSelectableRows boolean Enables the selectable rows feature. Use "true" to enable the selectable rows feature, and render checkboxes in the first column of each table row.
actions Action[] An array of objects where each object represents a button in the actions column. When there are more than 3 actions, the individual buttons collapse into a dropdown.
hasViewSelector boolean Enables the view selector feature. Use "true" to enable the view selector, and render the view selector below the table.
hasColumnSelector boolean Enables the column selector feature. Use "true" to enable the column selector, and render the column selector below the table.
hasDisplayDensity boolean Enables the display density feature.

Use "true" to enable the display density feature, and render the display density selector below the table.

The default display density is "comfortable."

The user can select "compact" to decrease the padding of the table's rows.

internationalization object Contains all the strings of text used in the table component. See internationalization for more details.

Links

Link properties
Property Type Description Notes
element string The element used for the link.

Expects either "a" or "button".

Use "a" when the link column should route to another part of the application.

Use "button" when the link column should perform an action on the page, e.g. opens a modal.

ariaLabel string The format of the aria-label to be used for the link. To use values from the row, insert #{key} anywhere into the string where "key" is the name of the data property to insert into the label, e.g. "Open row #{id}" NOTE: These are still regular strings, not javascript template strings, so use single/double quotes.
target string The string to be used in the link's target attribute. Required when element = "a". "_blank" opens in a new tab.
path string The string to be used in the link's routerLink attribute. One of either path or href is required when element = "a".
href string The string to be used in the link's href attribute. One of either path or href is required when element = "a".
action string The string to be emitted when the button is clicked. Required if element = "button". See Outputs for more information on what is emitted.

onSort properties
Output Type Description

Pagination

Pagination can be added to a table by nesting the bp-pagination component in the table component.

You can also provide a custom pagination component if needed by adding a pagination attribute to the nested component, e.g.: <myPagination pagination></myPagination>

Template Columns

Templates can be provided to specified columns to allow custom columns.

  1. Create a template in the component. This should either be a sibling or a child of the table component.
  2. Select the template in the component typescript.
  3. Pass the template into a column object of type "TEMPLATE".

Expandable Rows

When enabled, each row has an expandable row using a template passed in from the properties object.

  1. Create a template in the component. This should either be a sibling or a child of the table component.
  2. Select the template in the component typescript.
  3. Pass the template into the expandableRowsTemplate via the properties Input object.
