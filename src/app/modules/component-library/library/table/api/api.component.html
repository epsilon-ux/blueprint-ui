Inputs

Table inputs
Input Type Description Notes
data object[] An array of objects where each object represents a row in the table. Required.
isDataLoading boolean Indicates if the table data is loading.

If "true", the loading icon and "Loading data" message renders in the table.

If "false" and there is no data, the "No data available" message renders in the table.
properties object An object used to configure an instance of the table component.

Use the properties object to enable or disable configurable features.

See the Properties table for more information.

Properties

Table properties
Property Type Description Notes
caption string A sentence used for explaining the table to people using a screen reader. The text is only visible to screen readers. Required.
rowId string The key from the data to use as the identifier.
columns object[] An array of objects that define the columns in the table. Columns will render in the table in the same order as in this array. Required.
columns.key string The key from the inputted data to use for data in the corresponding column. Required.
columns.headerText string The text to display in the corresponding column header. Required.
columns.isColumnDisplayed boolean Indicates whether or not the column should be displayed by default. Use "true" if the column should be displayed by default.
columns.type enum ColumnType {
ID,
TEXT,
LINK,
DATE,
BOOLEAN,
INPUT,
STATUS,
ICON,
DROPDOWN
} 		The type of the column. This determines how to handle the content.
columns.template TemplateRef Pass in an ng-template that will be used for the content in this column. Required when column type is TEMPLATE.
columns.icon object An object containing information for when and how to render icons in an icon column. Required when column type is ICON.
columns.icon.color string The color of the icon. Expects either 'midnight' or 'warning'.
columns.icon.mapping object Maps icons to text. The key is the text that represents the icon, the value is the class name for the fontawesome icon.
columns.link.element string Expects either 'a' or 'button'. Use 'a' when the link column should route to another part of the application. Use 'button' when the link column should perform an action on the page, e.g. opens a modal.
columns.link.ariaLabel string The format of the aria label to be used for the link. To use values from the row, insert ${key} }anywhere into the string where 'key' is the name of the data property to insert into the label, e.g. 'Open row ${id}' NOTE: These are still regular strings, not javascript template strings, so use single/double quotes.
columns.link.target string The string to be used in the link's target attribute. Required when element = 'a'. "_blank" opens in a new tab.
columns.link.path string The string to be used in the link's routerLink attribute. One of either path or href is required when element = 'a'.
columns.link.href string The string to be used in the link's href attribute. One of either path or href is required when element = 'a'.
columns.link.action string The string to be emitted when the button is clicked. Required if element = 'button'. See Outputs for more information on what is emitted.
columns.statusIndicatorMapping object Maps the standardized class names for status indicators to the product specific text. Required when column type is STATUS. The available classes for mapping are incomplete-primary, incomplete-secondary, in-progress, warning, complete. For more information on status indicators see Zeplin.
search.hasSearch boolean Enables the search feature. Use "true" to enable search, and render the search component above the table.
sort.defaultSortedColumn string The key from the data that represents the column that is sorted by default / on page load. The column that is sorted by default cannot be hidden with the column selector.
hasSelectableRows boolean Enables the selectable rows feature. Use "true" to enable the selectable rows feature, and render checkboxes in the first column of each table row.
actions object[] An array of objects where each object represents a button in the actions column. When there are more than 3 actions, the individual buttons collapse into a dropdown.
actions.element string

Expects either a or button.

Use a when the action should route to another part of the application.

Use button when the action should perform an action on the page, e.g. opens a modal.
actions.text string The text to be used in the action button/link.
actions.ariaLabel string The format of the aria label to be used for the button/link. To use values from the row, insert ${key} anywhere into the string where 'key' is the name of the data property to insert into the label, e.g. 'Open row ${id}' NOTE: These are still regular strings, not javascript template strings, so use single/double quotes.
actions.class string The class(es) to be used on the action button/link

E.g. btn-outline-primary

If element = button, .btn will be included by default.
actions.target string The string to be used in the link's target attribute Required when element = 'a'. E.g. "_blank" opens in a new tab.
actions.routerLink string The string to be used in the link's routerLink attribute. One of either path or href is required when element = 'a'.
actions.href string The string to be used in the link's href attribute. One of either path or href is required when element = 'a'.
actions.action string The string to be emitted when the button is clicked. Required if element = 'button'. See outputs for more information on what is emitted.
actions.conditions (object | string)[] The conidtions used to determine if the action will render. Conditions is an array containing objects and strings. The condition objects have compare operator strings between them.
hasColumnSelector boolean Enables the column selector feature. Use "true" to enable the column selector, and render the column selector below the table.
hasDisplayDensity boolean Enables the display density feature.

Use "true" to enable the display density feature, and render the display density selector below the table.

The default display density is "comfortable."

The user can select "compact" to decrease the padding of the table's rows.
pagination.hasPagination boolean Enables the pagination feature.

Use "true" to enable pagination, and render the pagination below the table.

Pagination includes: the total number of results, a select that allows the user to select the number of rows to display per page, and pager buttons.

Outputs

Table outputs
Output Type Description
action object All outputs are emitted through the actions object. Actions occur when the user selects rows, clicks an element in the link column, and/or clicks an element in the actions column. When an action is triggered, the table emits an action object that contains an action and an id.
action.action string A string used to know what action occurred. This is specified in the Inputs (see Inputs & Properties).
action.id string The id of the row to which the action applies.
