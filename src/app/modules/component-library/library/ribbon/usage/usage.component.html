Ribbons provide important information to a user at a certain point of interaction. This information can appear depending on a user's actions, a system action, or the state of an item. Ribbons stay on the page until the user dismisses the message.

Ribbons are most commonly used above tables to give the user more information about the table items. The text content should be short, to the point, and not repetitive. Ribbons have a content area on the right side of the ribbon. This area can be used for adding different actions to the ribbon.

Variations

Informational ribbon

Informational ribbons provide additional topical information to the user without disrupting their workflow. These ribbons should appear near the related elements on the page.

This is an informational ribbon using content projection to add custom styled content

Success ribbon

This variation should be used for telling the user that their action was successful.

This is a success ribbon using content projection to add custom styled content, a related action, and a close button

Alert ribbon

This variation should be used for telling the user urgent information or alerting them that their action or inaction might have adverse effects.

This is an alert ribbon using content projection to add custom styled content, a related action, and a close button

Warning ribbon

This variation should be used for telling the user urgent information or warning them that their action or inaction resulted in an error.

This is a warning ribbon using content projection to add custom styled content, a related action, and a close button
