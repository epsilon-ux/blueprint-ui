The Epsilon product suite can be very complicated to learn how to use. Writing is one way we can guide our users through their product experience. For this reason, written content must be helpful, clear, and remain consistent across products.

Date & Time

There are a lot of dates and times within Epsilon products. When it comes to translation and internationalization, it is important to be consistent with our formats.

Date

Dates should always be written as numerals. This is for internationalization purposes.

Format

  • Month and day should always appear as two digits
  • Years should always appear as four digits
  • MM/DD/YYYY

Example

  • 12/1/2020
  • 1/31/2019

Time

Times can appear by themselves or attached to a date.

Format

  • Single digit hours should appears a single digit
  • Am / pm designation should appear immediately after the time; do not add a space
  • Time zone can be abbreviated or spelled out, depending on the situation
    • If abbreviated, time zone abbreviation should be in all-caps
  • Do not show seconds
  • HH:MMam/pm time zone

Example

  • 3:30pm EST
  • 12:20am UTC

Date & time

Dates and times often appear together in our products.

Format

  • Date should always appear first
  • Time should go directly after date separated by one space
  • Do not show seconds
  • MM/DD/YYYY HH:MMam/pm time zone

Example

  • 12/21/2020 11:32am EST
  • 01/31/2020 12:05pm UTC

Numbers

There are a lot of numbers in Epsilon products. Here are some guidelines when writing with numbers:

  • Spell out the number when it begins a sentence
    • Ten objects are being deployed.
  • Spell out the number when it would be weird to not. Generally, this means spelling out ordinals:
    • Third-party
    • Second step
    • All-in-one
  • Otherwise it is completely acceptable to write the number using numerals
    • Enter up to 40 email addresses & customer keys to test your program with

Numbers with commas

Use commas in numbers with four or more digits. Adding commas makes the numbers easier to read out and scan.

Example

  • 999
  • 1,000
  • 150,000
  • 1,500,000

Abbreviated numbers

It is acceptable to abbreviate large numbers when space is tight. This generally is used on charts and dashboards.

Example

  • 1k
  • 20k
  • 1m

Money and currency

When writing about money, US dollars, or any other currencies use the currency sign before the amount and include decimals if there is a cent amount.

Example

  • $20
  • $20.15
  • €1.34
  • ¥5

Telephone numbers

Use dashes between number groups. Don't add spaces or parentheses.

Example

  • 800-867-5309
  • +1-800-867-5309

Button Text

Button text should be short, clear, and always contain an action. Title case should be used for button text, meaning every word should be capitalized, except smaller words (check Case Guidelines).

Guidelines

  • Use action words and keep it short
    • Ampersands (&) may be used in buttons to shorten the length
  • Text may be in answer to a question within the interface, such as an 'Are you sure?' question in warning modals
    • For example, 'Yes, Continue' or 'No, Undo' button text
  • For primary buttons that create new objects use the format 'Create Object'

Example

  • Create Event
  • Log In
  • Save
  • Save & Exit
  • Yes, Continue
  • No, Discard Changes

Field Text and Placeholder Text

Field labels and placeholder text are crucial parts of form elements. They describe to the user the purpose of the text field, dropdown, or any other input field. All input fields should include a label and placeholder text.

Field text guidelines

  • Be short, precise, and informative
  • Fields that are not required to complete the action should be labeled as 'Optional'
  • Use title case (Case Guidelines)

Placeholder guidelines

  • Placeholder text can be longer than the field label, as long as it is descriptive and to the point
  • Depending on the situation, placeholder text can address the user (as 'you') or side-step the user entirely
    • If describing a large direct action, such as a create flow, use 'you'
    • If describing a small selection or input, such as a filter, don't address the user
  • Use sentence case (Case Guidelines)

Example

  • Enter a point amount to reward
  • Select a category
  • Search by keyword
  • Select what kind of marketing promotion you would like to create

Abbreviations

Abbreviations and acronyms are a handy way to shorten long and cumbersome phrases. But, users may not understand or recognize these short-hand references at first. To fix this, spell out the full term or phrase, accompanied with its abbreviation in a parentheses, the first time it appears on a page or in an object. After this, you can use the abbreviated version of the term.

Guidelines

  • Always spell out the full phrase on first use
  • Don't spell out file extensions
  • If the abbreviation or acronym is widely well known, there is no need to spell it out
    • API, HTML, CAPTCHA, ID, certain time zones
  • Time zones
    • When possible, spell out time zones, for both international and continental United States
    • When accompanied with a selected date and time, use the time zone abbreviation

Example

  • Coordinated Universal Time (UTC)
  • Eastern Standard Time (EST)

Positive Language

Use positive language over negative language whenever possible. This sets a more pleasant tone and tends to be shorter and more to the point.

Do

This action can only be used in programs that contain its associated event.

Please select a parent to view your programs.

Don't

This action can't be used in programs that don't contain its associated event.

Programs can't be shown because there is no parent selected.

Pronouns

Although, it is uncommon to refer to users or groups of users within our products, we have some very simple rules to follow when talking to or about people.

Guidelines

  • Avoid gendered pronouns and terms
    • He, she, girls, guys
  • Use neutral terms, such as 'they' or 'you' or avoid addressing the user altogether
  • Only use a gendered pronoun if it is specifically stated that that is the user's chosen pronoun

Ampersands

The ampersand (&) represents the word 'and'. It originated as a ligature of the Latin word for and — 'et'.

Guidelines

  • Do not use ampersands in paragraphs of text
  • Use only in button labels, headers, tabs, and toggles

Example

  • Save & Continue
  • Voice & Tone

File Extensions

If using the file extension within a sentence, capitalize the acronym. If using as a file suffix, lowercase the extension.

Example

  • PDFs
  • CSVs
  • filename.jpg
  • filename.png
