Heading 1

This is an example of a Heading 1

Guidelines

  • Text set in heading 1 should always be in title case (see Voice & Tone for case guidelines)
  • There should always be one heading 1 on a page, but no more than one

Common uses

  • Page titles
  • Object names
  • Modal headers

Heading 2

This is an example of a Heading 2

Guidelines

  • Text set in heading 2 should always be in title case (see Voice & Tone for case guidelines)
  • Text set in heading 2 should be short and to the point
  • There is no limit to the number of heading 2 styles that can be on a page

Common uses

  • Section headers
  • Primary tabs

Heading 3

This is an example of a heading 3

Guidelines

  • Text set in heading 3 should always be in sentence case (see Voice & Tone for case guidelines)
  • Text set in heading 3 should be short and to the point
  • There is no limit to the number of heading 3 styles that can be on a page
  • Heading 3 text should follow or be nested under heading 2 text

Common uses

  • Sub-headers
  • Sub-sections within heading 2 sections
  • List headings

Heading 4

This is an example of a heading 4

Guidelines

  • Text set in heading 4 should always be in sentence case (see Voice & Tone for case guidelines)
  • There is no limit to the number of heading 4 styles that can be on a page
  • Heading 4 text should follow or be nested under heading 3 text

Common uses

  • Secondary sub-headers
  • Sub-sections within heading 3 sections

Lead Paragaph

This is an example of a lead paragraph

Guidelines

  • Lead paragraphs should always be in sentence case (see Voice & Tone for case guidelines)
  • Lead paragraphs should follow a heading 1 or 2
  • Lead paragraphs should not follow a heading 3 or 4
  • Lead paragraphs should be used sparingly

Common uses

  • Overview summaries
  • Primary tabs
  • Large text labels & descriptions
    • These should be used on pages that have a simple layout and singular focus, such as full-page modals

Emphasized Body

This is an example of emphasized body

Guidelines

  • Emphasized body can be set in title or sentence case depending on the use
  • Emphasized body is used for labels, selected items, or text that needs to be set apart from the rest of the body copy

Common uses

  • Key value pairs
  • Table column headers
  • Side navigation selected pages
  • Primary navigation expanded dropdowns
  • List headings
  • Expand and collapse tree structures

Primary Body

This is an example of primary body

Guidelines

  • Primary body can be set in title or sentence case depending on the use
  • Primary body is the most commonly used text style
  • Primary body is used for all text, notes, inputs, and information essential to the product

Common uses

  • Key value pairs
  • Table content
  • Informational text
  • Inputs and selections
  • Radio button and checkbox labels
  • Side navigation items
  • Secondary tabs

Muted Body 1

This is an example of muted body 1

Guidelines

  • Muted body 1 can be set in title or sentence case depending on the use
  • Muted body 1 is used for all text that is not essential or if the text should be de-emphasized

Common uses

  • Key value pairs
  • Informational text
  • Help text
  • Disabled inputs and selections
  • Disabled radio button and checkbox labels
  • Disabled secondary tabs
  • Unselected toggles

Muted Body 2

This is an example of muted body 2

Guidelines

  • Muted body 2 can be set in title or sentence case depending on the use
  • Muted body 2 is used for text that is not essential and should be de-emphasized to avoid drawing attention away from the main copy or focus on the page

Common uses

  • N/A notations
  • Dates, such as 'as of' dates or date range limitations
  • Directional text, such as drag and drop notes

Muted Body 3

This is an example of muted body 3

Guidelines

  • Muted body 3 can be set in title or sentence case depending on the use
  • Muted body 3 is the least common type style
  • Muted body 3 is used for text that is not essential and should be de-emphasized, but also needs to have heavier presence on the page for hierarchial reasons

Common uses

  • Non-tabular column headers
  • Secondary labels

Error Messaging

This is an example of an error message

This is an example of an error message with an icon

Guidelines

  • Error messages should always be in sentence case (see Voice & Tone for case guidelines)
  • Error messages are in tomato; if there is an accompanying icon, the icon should also be in tomato
  • Error messages should be short, to the point, and provide actionable information
  • Error messages should appear below the field element that it is referring to

Action Text

This is an example of a link

This is an example of a link with an icon

Guidelines

  • Action text shorter than three words should be set in title case
  • Action text longer than three words should be set in sentence case
  • Action text is 12px, semibold, and ultramarine; if there is an accompanying icon, that icon should also be 12px and ultramarine
