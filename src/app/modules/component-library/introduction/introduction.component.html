Usage

Please do not override or customize Blueprint components. If Blueprint components are overridden or customized in products, we will be unable to safely update the epsilon-blueprint component library without introducing breaking changes in the products. If you experience any issues integrating Blueprint components, please reach out to the UX team.

Setup

Reach out to the UX Team when setting up the application's base template and architecture to work best with Blueprint, align to other products, and ensure scalability.

This package has a few dependencies that are not included in the package itself. They are as follows:

  • Angular 8
  • ng-select

The package also relies on having the following imports in the application's index.html.

Bootstrap v4

Font Awesome 5

Open Sans

Installation

Once you have all these, run npm install epsilon-blueprint to install the package to node_modules. Once the package is installed, import the stylesheets from the package.

In the application's styles.scss file, add the following import statement. @import "~epsilon-blueprint/styles";

When global variables are needed in a scss file, add the following import statement. @import "~epsilon-blueprint/styles/global-variables";
