Usage

Please do not override or customize Blueprint components. If Blueprint components are overridden or customized in products, we will be unable to safely update the epsilon-blueprint component library without introducing breaking changes in the products. If you experience any issues integrating Blueprint components, please reach out to the UX team.

Project setup

Reach out to the UX Team when setting up the application's base template and architecture to work best with Blueprint, align to other products, and ensure scalability.

Dependencies

The epsilon-blueprint package has a few dependencies that are not included in the package itself. They are as follows:

  • Angular 8
  • ng-select

Imports

The package also relies on having the following imports in the application's index.html.

Bootstrap v4
Font Awesome 5
Open Sans

Automated code quality tools

All applications should use automated code quality tools to help avoid errors, provide real-time feedback, reduce the amount of human code review needed, allow code reviewers to focus on providing higher-level feedback, and enforce best practices and coding standards.

stylelint
  1. Run npm install stylelint stylelint-config-standard --save-dev
  2. Run npm install stylelint-order --save-dev
  3. Run npm install stylelint-scss --save-dev
  4. Add the stylelint config to the root of the project
  5. In the application's package.json file, add the following script: "stylelint": "stylelint '**/*.scss'"
ESLint
  1. Run npm install eslint --save-dev
  2. Run npm install @typescript-eslint/parser --save-dev
  3. Run npm install @typescript-eslint/eslint-plugin --save-dev
  4. Run npm install @angular-eslint/eslint-plugin --save-dev
  5. Add the ESLint config to the root of the project
  6. Add the ESLint ignore to the root of the project
  7. In the application's package.json file, add the following script: "eslint": "eslint 'src/**/*.ts'"

Installation

Once project setup is complete, install epsilon-blueprint.

  1. Run npm install epsilon-blueprint
  2. In the application's styles.scss file, add the following import statement: @import "~epsilon-blueprint/styles";
  3. When global variables are needed in a scss file, add the following import statement: @import "~epsilon-blueprint/styles/global-variables";
